# Contributing to AI-Staff-HQ

First off, thank you for considering contributing to AI-Staff-HQ! It's people like you that make this project such a powerful tool.

This document provides guidelines for contributing to the project. Please read it carefully to ensure that your contributions are accepted.

## Code of Conduct

This project and everyone participating in it is governed by the [AI-Staff-HQ Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## How Can I Contribute?

There are many ways to contribute to AI-Staff-HQ, from improving the documentation to creating new specialists. Here are some of the ways you can get involved:

### **Reporting Bugs**

If you find a bug in the system, please open an issue on our GitHub repository. When you are creating a bug report, please include as many details as possible. Fill out the required template, the information it asks for helps us resolve issues faster.

### **Suggesting Enhancements**

If you have an idea for a new feature or an enhancement to an existing one, please open an issue on our GitHub repository. We are always looking for ways to make AI-Staff-HQ better.

### **Pull Requests**

If you want to contribute code to the project, you can do so by submitting a pull request. Before you submit a pull request, please make sure that you have done the following:

1.  Fork the repository and create your branch from `main`.
2.  Make sure that your code adheres to the project's coding standards.
3.  Add tests for your code.
4.  Make sure that all tests pass.
5.  Update the documentation to reflect your changes.

### **Creating New Specialists**

One of the most valuable ways you can contribute to AI-Staff-HQ is by creating new specialists. To ensure that new specialists are consistent with the existing ones, we have created a template that you can use.

To create a new specialist, please use the `templates/persona/new_specialist_submission.md` template. This template will guide you through the process of creating a new specialist, including defining their role, skills, and KPIs.

## Styleguides

### **Git Commit Messages**

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally after the first line.

### **YAML Styleguide**

- Use 2 spaces for indentation.
- Use hyphens for sequences.
- Use double quotes for strings.

We look forward to your contributions!
