# 🧠 Research Notes

> _A place to save research your specialists can read and reuse._

## Why This Folder Exists

Your specialists do better work when they have good facts to draw from. This folder is where you keep that research. Save a note once, and any specialist can use it later. No extra API calls, no re-searching the same thing twice.

This folder starts mostly empty. You fill it as you go.

## Suggested Folders

Make a subfolder when you start collecting a kind of research. These four cover most needs:

- **`market-intelligence/`** — Market size, trends, and what customers want. Useful for the Market Analyst, Brand Builder, and Chief of Staff.
- **`creative-trends/`** — Design styles, culture shifts, and what's popular now. Useful for the Creative Strategist and Copywriter.
- **`technical-insights/`** — Tools, methods, and tech that's worth using. Useful for the Automation Specialist and Software Architect.
- **`competitive-analysis/`** — What other products and brands are doing. Useful for the Market Analyst and Brand Builder.

You can add your own folders any time.

## Naming Your Files

Use a clear name so you can find it later:

```
[date]_[topic].md

Example:
2026-05-30_ai-adoption-trends.md
```

## A Simple Note Template

Copy this when you add a note:

```markdown
# [Title]

## Summary
Two or three sentences. The key point at a glance.

## Why It Matters
How this helps your work or your plans.

## Key Findings
- Bullet points with the main facts.

## Source
Where it came from, and how much you trust it.

## Added
Date, and which specialists this helps.
```

## How Specialists Use It

When a specialist needs facts, point it at a note in this folder. For example:

> "Market Analyst, read `market-intelligence/2026-05-30_ai-adoption-trends.md` and use it to shape the positioning."

Keep your best notes current, and drop ones that go stale. A small, fresh set beats a big, outdated pile.
