# 🚀 Automated Project Scoping Autonomous Workflow

> An autonomous workflow for automatically generating a project brief from a high-level goal.

## 🎯 Workflow Overview

**Purpose**: To automate the strategic front-end of a project by generating a comprehensive project brief with minimal user input.  
**Trigger**: A user request for a new project with a high-level goal.  
**Facilitation**: This is an autonomous workflow, primarily driven by the Chief of Staff.  
**Deliverables**: A completed `Project Brief Template` for user approval.

---

## 🔄 Workflow Steps

### **Phase 1: Goal Ingestion (Chief of Staff)**

1.  **Receive Goal**: The Chief of Staff receives a high-level project goal from the user.
2.  **Activate Specialists**: The Chief of Staff activates the `Market Analyst` and `Brand Builder` to begin the project scoping process.

### **Phase 2: Research and Analysis (Market Analyst)**

1.  **Market Research**: The `Market Analyst` conducts market research to identify the target audience, market trends, and competitive landscape.
2.  **Audience Analysis**: The `Market Analyst` provides a detailed analysis of the target audience's needs, pain points, and motivations.

### **Phase 3: Strategic Positioning (Brand Builder)**

1.  **Brand Strategy**: The `Brand Builder` develops a brand strategy and positioning for the project based on the Market Analyst's research.
2.  **Messaging Architecture**: The `Brand Builder` creates a messaging architecture that will guide the project's communication.

### **Phase 4: Project Brief Generation (Chief of Staff)**

1.  **Synthesize Information**: The Chief of Staff synthesizes the information from the `Market Analyst` and `Brand Builder`.
2.  **Generate Project Brief**: The Chief of Staff generates a completed `Project Brief Template` with the synthesized information.

### **Phase 5: User Approval**

1.  **Present Project Brief**: The completed `Project Brief Template` is presented to the user for approval.
2.  **User Decision**: The user can either approve the project brief or request changes.

### **Phase 6: Workflow Completion**

1.  **Approval**: If the user approves the project brief, the workflow is complete.
2.  **Next Steps**: The user can then initiate the next phase of the project, such as creative development.
