# Specialist Examples

This folder contains completed specialist YAML files drawn from real engagements. Each file demonstrates how the templates evolve once a role is fully deployed.

- Focus on how activation prompts, guardrails, and integration notes reflect actual usage.
- Treat these as reference points for structure and depth—not as roles to reuse verbatim.
- Pair each specialist with the accompanying `notes-on-creation.md` entry to understand design decisions and trade-offs.

Start by reading `notes-on-creation.md`, then inspect the YAML files to see how strategy, activation, and escalation guidance align with real work.
