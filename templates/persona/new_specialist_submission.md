# New Specialist Submission

> A template for submitting a new specialist to the AI-Staff-HQ.

## Specialist Overview

**Name**: [Specialist Name]
**Department**: [Creative, Strategy, Technical, Kitchen, Personal, Specialized, Commercialization]

**Role Description**: [A brief, one-sentence description of the specialist's role.]

**Core Function**: [A more detailed description of the specialist's core function and how they contribute to the AI-Staff-HQ.]

## Specialist Configuration

```yaml
name: [Specialist Name]
department: [Department]
role: [Role Description]

skills:
  - [Skill 1]
  - [Skill 2]
  - [Skill 3]

activation_patterns:
  - "Act as my [Specialist Name]..."
  - "[Specialist Name], I need you to..."

quality_standards:
  - [Quality Standard 1]
  - [Quality Standard 2]

kpis:
  - kpi: [KPI 1]
    target: [Target for KPI 1]
  - kpi: [KPI 2]
    target: [Target for KPI 2]
```

## Submission Checklist

- [ ] The specialist's role is clearly defined and does not overlap with existing specialists.
- [ ] The specialist's skills are well-defined and relevant to their role.
- [ ] The specialist's activation patterns are clear and easy to use.
- [ ] The specialist's quality standards are specific and measurable.
- [ ] The specialist's KPIs are quantifiable and aligned with their role.
- [ ] The YAML configuration is valid and follows the project's style guidelines.
